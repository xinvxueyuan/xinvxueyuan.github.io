# Product

## Register

brand

## Platform

web

## Users

主要用户是阅读中文技术文章、关注开源项目的读者。他们可能在桌面端深度阅读教程，也可能在移动端快速查看新文章。站长本人通过 Git 提交 Markdown 发文，不需要维护数据库或 CMS。

## Product Purpose

xinvStar 是一个以技术写作为主干的长期个人博客。MVP 成功意味着读者能从首页发现文章，并在桌面或移动端舒适阅读；历史 URL、订阅入口和旧站附加功能不在本次重写范围内。

## Positioning

它不是通用博客模板，而是一座把代码与学习轨迹编排成同一片星图的个人问讯站。

## Conversion & proof

- 唯一主要行动是浏览并阅读最新文章。
- 访客十秒后应记住：这里既能认真读技术，也能遇见一个具体的人。
- 信念路径：先确认内容值得读，再确认站点易于浏览，最后愿意收藏文章或继续访问。
- 现有证明是已经积累的 Markdown 文章与持续写作记录。

## Brand Personality

好奇、真诚、灵动。整体应像夜晚书桌旁一台仍亮着的个人终端：有技术秩序，也有星光、音乐与轻微的浪漫感。体验可以带一点日系圆润与二次元氛围，但不能依赖受版权角色替品牌说话。

## Anti-references

- 不做无人物痕迹、套壳即可复用的 SaaS 落地页。
- 不做由同尺寸图标卡片、玻璃拟态、紫色渐变和巨大指标组成的 AI 模板站。
- 不复制现站首页的控件墙、数百个可聚焦标签和层层嵌套卡片。
- 不使用未经确认授权的动漫、游戏角色作为主视觉。
- 不把个人博客伪装成杂志排版练习或终端模拟器。

## Design Principles

1. 阅读先于装饰：无 JavaScript、第三方失败或低速网络下，正文和导航仍然完整。
2. 个性集中表达：原创主视觉和主题形成可识别舞台，但不挤占阅读区域。
3. 内容保持简单：文章源文件是唯一内容契约，路由与派生功能可以以后重建。
4. 渐进增强：互动能力必须下沉到最小客户端边界，失败时不拖垮页面。
5. 原创胜过借用：以 xV 星轨、知识节点和站长自己的内容建立识别，而不是借角色素材获得氛围。

## Accessibility & Inclusion

目标达到 WCAG 2.2 AA；关键页面 axe serious/critical 为零，Lighthouse Accessibility 不低于 0.90。必须支持键盘导航、清晰焦点、Skip Link、语义 landmark、单一 H1、可靠表单标签、减少动态效果、系统主题以及不依赖颜色传达信息。
