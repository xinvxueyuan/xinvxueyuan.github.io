# 当前视觉与功能基线

## 采集范围

使用 headless Chrome 直接访问线上站点，覆盖 390×844、768×1024、1440×900，亮色、暗色和 reduced-motion。共保存 18 张截图到 `artifacts/baseline/screenshots/`，并验证首页、本地 Preview、普通文章、代码、Mermaid、KaTeX、归档、搜索、移动导航、相册、日记、追番、项目展示与真实 404。

标签和分类不是独立页面路径，而是 `/archive/?tag=...` 与 `/archive/?category=...`；RSS、Atom 和 Sitemap 是机器可读端点，不以视觉页替代检查。

## 信息架构与交互

- 顶部导航按主页、归档、链接、我的、关于、其他分组；移动端收拢到菜单按钮。
- 首页同时承载个人介绍、标签/分类入口、文章分页、日历、音乐播放器和多组侧栏组件，信息密度高。
- 文章页保留目录、元信息、上下篇、分享、评论入口和 Markdown 扩展能力。
- 搜索由 Pagefind 客户端索引驱动；构建实际索引 39 页、4480 词。
- 主题、壁纸来源、布局和音乐均有客户端状态；暗色主题实际切换到 `github-dark`。

## 视觉特征与品牌识别

- 大幅随机横幅、半透明卡片、圆角、阴影和强调色共同构成主要识别点。
- 二次元头像、日文短句、背景图与悬浮音乐控制是强品牌元素。
- JetBrains Mono、KaTeX 字体和代码块卡片构成技术博客阅读风格。
- 三栏桌面布局在平板/手机逐步收拢；采样页未发现横向溢出。

## 值得保留

- 文章 URL、阅读层级、目录和 Markdown 渲染语义。
- 亮/暗主题、移动导航、搜索、归档筛选和内容专页入口。
- 品牌头像、横幅氛围、卡片语言和音乐/壁纸的“个性站点”定位。
- 代码复制、Mermaid SVG、KaTeX 和图片查看等内容增强。

## 可重设但不得破坏功能

- 首页组件数量、侧栏密度、悬浮控件位置和卡片装饰可在设计阶段精简。
- 音乐、壁纸、日历等应拆成最小 Client Component，不应迫使整页客户端化。
- 标签与分类可改善视觉呈现，但查询 URL 和筛选语义必须兼容。

## 当前问题与不确定项

1. 首页 DOM 有 500 余个可聚焦元素，标签云、分类和音乐控件造成较高交互密度。
2. 搜索面板和移动菜单的语义结构不够稳定；抽样时未观察到 `dialog` landmark。
3. 线上 HTML 未发现 canonical，属于 SEO 基线缺口，不应被迁移复制为目标行为。
4. `header/nav/footer/aside` 语义元素缺失，视觉区块主要由普通容器实现。
5. 无 JavaScript 浏览器仿真因工具导航超时未形成截图；构建 HTML 和直接响应仍包含完整文章正文，交互降级需在迁移验收时单独自动化。

原始浏览器摘要见 `artifacts/baseline/route-checks/browser-observations.json`。
