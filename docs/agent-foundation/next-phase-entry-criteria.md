# 下一阶段入口条件

## 判定

**READY WITH RISKS：可以进入自主设计阶段，但必须先由用户验收本报告。**

这不代表允许编写 Next.js 或发布生产；下一阶段只能先开展产品/视觉/架构设计，并继续使用新的隔离 Worktree。

## 已完成

- 隔离 Worktree、专用分支、恢复 Commit、项目代理规则和非部署 CI 已建立。
- 迁移所需 Skills 已审计、分级、固定；C 级 `impeccable` 未启用。
- 旧站仓库、依赖、内容、外部服务、构建产物、路由、SEO、视觉、性能和可访问性基线已生成。
- 真实安装、Astro check、只读 TypeScript/ESLint、基础测试和生产构建均有退出码与日志。
- 线上 211 条路由记录、37 篇发布文章、截图与浏览器交互证据已保存。
- 迁移验收矩阵与项目级 Meta Skill 草案已生成并静态校验。

## 未完成（留给设计/实现阶段）

- 未实施任何 Next.js 页面、组件、内容管线或部署配置。
- 未验证独立内容仓库的私有权限和真实同步；当前跟踪内容足以设计。
- 未获得稳定 Lighthouse 分类分数或完整 axe 报告；Preview 阶段必须补齐。
- 未确认生产部署端动态 OG 图片的真实行为。
- 未创建 Vercel Preview、未触碰生产/DNS/正式域名。

## 会阻止迁移实现或发布的问题

1. 设计规格或验收矩阵未获批准时，不得进入实现。
2. 内容仓库等价同步方案在删除旧管线前必须通过隔离集成测试。
3. Preview 未补齐 URL 全量回归、Lighthouse/axe 和回滚验证时，不得 Promote。
4. 用户没有对最终 Preview 明确验收时，不得生产发布。

## 可在迁移过程中解决的风险

- 旧站独立 tsc/ESLint 失败：新 Next.js 工具链从严格绿基线重建。
- canonical、Landmark、Skip Link、Heading/Label 等旧站缺口：作为新站改进目标。
- 横幅导致的 CLS、较大的图片/字体和客户端交互密度：设计与性能阶段处理。
- OG 生成、外部数据缓存、壁纸/追番 fail-open：在架构规格中明确。

## 用户现在唯一需要决定的事项

是否接受本前置基础设施与 `READY WITH RISKS` 判定，并批准下一阶段开始“自主设计”（不是编码或上线）。批准后，Meta Skill 会先提出设计方向供用户验收，再进入实施计划。
